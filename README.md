# ImitationWorld
Unity 엔진을 사용해 개발한 3D 기반 오픈월드 서바이벌 게임 프로젝트입니다.

---

## 📌 프로젝트 개요

- **장르**: 3D 서바이벌 오픈월드
- **개발 형태**: 개인 프로젝트
- **개발 기간**: 2025.05 ~ 2025.06 (약 6주)
- **목적**: 환경 오브젝트와의 상호작용, 인벤토리 시스템, 아이템 제작 시스템, 스킬 시스템, 아군/적군 몬스터 AI 구현을 연습하기 위해 제작하였습니다.

---

## 🎮 게임 설명

- 플레이어는 환경 오브젝트를 통해 아이템을 수집하고 제작을 통해 새 아이템을 만들어내어 성장합니다.
- 필드 위 몬스터와 전투할 수 있으며, 몬스터를 포획하여 동료로써 함께 다닐 수 있습니다.
- 게임의 핵심 시스템은 아이템 제작과 전투입니다.

## ✨ 핵심 구현 내용

- FSM(State Machine) 기반 캐릭터 상태 관리
- 이벤트를 기반으로 한 UI 업데이트

## 🔍 문제 해결

### 문제: 오브젝트(몬스터, 이펙트, 아이템 등) 생성 시 프레임 드랍 발생
- **원인**: Instantiate / Destroy 반복으로 인한 GC 발생
- **해결**: Object Pooling 구조 도입
- **결과**: 평균 FPS 안정화 및 성능 개선

### 문제: 아이템 레시피를 하드코딩으로 관리 시 코드 복잡도가 증가하고 확장이 어려움
- **원인**: 레시피 구조가 코드에 직접 박혀 있어 수정/추가 시 복잡해짐
- **해결**: 레시피를 ScriptableObject로 분리/관리하고 CraftObject가 레시피 데이터를 참조하도록 구조 변경
- **결과**: 레시피 추가/수정이 데이터 작업으로 단순화되고 유지보수성이 향상됨

### 문제: Terrain에 배치한 나무/돌이 물리 상호작용(피격/파괴) 대상이 되지 않음
- **원인**: Terrain의 트리/디테일 오브젝트는 게임오브젝트가 아니어서 콜라이더/스크립트를 붙일 수 없음
- **해결**: 상호작용 가능한 Prefab을 Terrain 랜덤 위치에 스폰하고, SampleHeight로 지면 높이 보정 + 노멀 기준 회전 정렬
- **결과**: 환경물과 타격/파괴 상호작용 가능, 지형 경사에 맞게 배치되어 자연스러운 배치 확보

### 문제: 적/아군 로직을 분리하면 중복 코드가 증가하고 유지보수 비용이 커짐
- **원인**: 행동 로직이 거의 동일한데도 진영에 따라 별도 구현하려는 구조
- **해결**: 동일 로직을 공유하고 진영 플래그(AllyEnemyConversion)와 태그 분기로 행동 차이를 처리
- **결과**: 몬스터 추가 시 로직 재사용성 상승, 유지보수 범위 축소

### 문제: 인벤토리 변경 시 UI 반영이 늦거나 누락될 가능성
- **원인**: UI를 매 프레임 폴링하거나, 변경 시점과 UI 갱신 타이밍이 분리되어 동기화가 깨질 수 있음
- **해결**: OnInventoryChanged 이벤트를 기준으로 UI 즉시 갱신
- **결과**: UI 반응 속도 개선, 불필요한 Update 폴링 제거로 성능/정합성 개선

### 문제: 체격 차이가 큰 몬스터에게 스킬이 맞지 않음
- **원인**: 스킬/투사체 발사 방향이 대상 기준점(발바닥 높이 등)으로 계산되어 높이 보정이 부족함
- **해결**: 투사체 방향을 대상 NavMeshAgent 높이의 1/2 지점으로 보정하여 조준 높이 개선
- **결과**: 대형 몬스터 대상 스킬 적중률 개선
